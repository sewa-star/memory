# 📋 작업 브리프

**원 명령:** 현재 로컬 두뇌로 파일을 입수할 때, 00_Raw/00_Raw/YYYY-MM-DD 형태로 '00_Raw' 폴더가 이중으로 겹쳐서 생성되는 문제가 발생하고 있습니다.

현재 내 로컬 두뇌의 기준 디렉토리(루트) 자체가 이미 00_Raw로 설정되어 있습니다. 따라서 앞으로 파일을 입수하고 경로를 지정할 때는 절대로 앞에 00_Raw/를 붙이지 마세요.

앞으로 저장할 대상 폴더는 무조건 YYYY-MM-DD 형태(예: 2026-05-13)로만 지정해서 저장 및 자동 푸시를 진행하도록 로컬 두뇌 경로 설정 규칙을 영구적으로 수정해 줘.

## 요약
로컬 두뇌의 파일 저장 경로에 중복 폴더(00_Raw/00_Raw/)가 생성되는 치명적인 시스템 오류를 해결해야 합니다. 현재 기준 디렉토리가 이미 00_Raw이므로, 파일 저장 시 절대 00_Raw/를 추가하지 않고, 대상 폴더는 무조건 YYYY-MM-DD 형식으로만 지정하여 경로 설정 규칙을 영구적으로 수정해야 합니다.
