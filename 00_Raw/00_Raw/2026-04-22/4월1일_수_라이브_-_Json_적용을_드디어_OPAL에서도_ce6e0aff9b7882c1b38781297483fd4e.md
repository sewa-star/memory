# 4월1일 수(라이브)- Json 적용을 드디어 OPAL에서도

제1장: JSON이란 무엇인가?

JSON(JavaScript Object Notation)은 컴퓨터와 컴퓨터, 혹은 사람과 AI가 대화할 때 사용하는 데이터 규격입니다.

쉽게 말해, 우리 공장을 이렇게 지어줘!라고 적어둔 디지털 설계도와 같습니다.

1. JSON의 기본 생김새: 이름과 값
JSON은 항상 "이름": "값"의 쌍으로 이루어집니다.
- 예: "title": "K-Video Creator" -> (해석: 이 설계도의 제목은 'K-Video Creator'야.)

---

제2장: 공장의 3대 핵심 요소

우리가 본 설계도에는 세 가지 중요한 섹션이 있었습니다.

1. Nodes (작업대)
공장 안에 놓인 기계들입니다. 각 기계는 자기만의 역할이 있습니다.
- ID: 기계의 고유 번호 (예: 220fa...)
- Type: 기계의 종류 (예: text-3-flash는 글쓰기 기계, video-veo-3-1은 영상 제작 기계)
- Configuration: 기계의 세부 설정 (예: 한국적인 느낌을 듬뿍 넣어줘!)
1. Edges (컨베이어 벨트)
기계와 기계를 연결하는 선입니다. 데이터가 어디서 어디로 흘러가는지 결정합니다.
- From/To: A 기계에서 만든 시나리오를 B 기계로 보내!
- Context: 단순한 물건이 아니라, 앞 단계 AI가 생각한 맥락(기억)까지 통째로 다음 기계에 전달합니다.
1. Metadata 및 Assets (인테리어와 창고)
- Metadata: 공장의 벽지 색깔이나 버튼 모양 같은 디자인 설정입니다.
- Assets: 영상 제작에 필요한 유튜브 링크나 이미지 같은 외부 재료들입니다.

---

제3장: 실제 흐름 들여다보기 (Veo 3.1 공장)

우리가 만든 비디오 생성 앱은 다음과 같은 순서로 작동합니다.

- 1단계: User Input (사용자의 아이디어를 받습니다. 예: 서울의 밤을 그려줘)
- 2단계: Generate 1 (아이디어를 멋진 영상 시나리오로 확장합니다.)
- 3단계: Generate 2 (시나리오를 바탕으로 실제 고화질 영상을 구워냅니다.)
- 4단계: Output (완성된 영상을 우리에게 보여줍니다.)

---

제4장: JSON 읽는 법 요약

설계도를 읽을 때 이것만 기억하세요!

1. { } (중괄호): 하나의 거대한 공장 전체를 묶어줍니다.
- [ ]  (대괄호): 비슷한 기계들이나 연결선들을 모아놓은 리스트입니다.
1. , (쉼표): 자, 다음 항목은 이거야라고 알려주는 구분자입니다.
