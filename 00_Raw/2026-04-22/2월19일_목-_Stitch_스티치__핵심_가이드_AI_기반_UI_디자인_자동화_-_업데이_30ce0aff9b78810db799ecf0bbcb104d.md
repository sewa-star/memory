### Stitch(스티치) 핵심 가이드: AI 기반 UI 디자인 자동화

텍스트(글)만으로 UI 디자인과 프론트엔드 코드를 자동 생성해 주는 도구인 **Stitch**의 8가지 핵심 기능을 학습합니다.

### 🎯 학습 목표

- Stitch의 기본 구조인 '프로젝트(Project)'와 '스크린(Screen)'의 개념을 이해합니다.
- AI를 활용해 화면을 생성하고 수정하는 명령어의 올바른 사용법과 주의사항을 숙지합니다.

## 💡 필수 용어 사전

학습에 앞서 가장 기본이 되는 두 가지 핵심 용어를 알아둡니다.

| **용어** | **개념** | **비유** |
| --- | --- | --- |
| **Project (프로젝트)** | UI 디자인과 프론트엔드 코드를 담는 **최상위 컨테이너** | 여러 장의 그림이 묶인 **'스케치북'** |
| **Screen (스크린)** | 프로젝트 내부에 존재하는 **개별 UI 화면** | 스케치북 안의 **'낱장 페이지'** |

---

## 📖 제1장. 프로젝트(Project) 관리하기

새로운 작업 환경을 만들고 전체 목록을 관리하는 기본 명령어들입니다.

| **명령어 (함수명)** | **기능 설명** | **활용 예시** |
| --- | --- | --- |
| **`create_project`** | **프로젝트 생성:** 완전히 새로운 Stitch 프로젝트를 만듭니다. | "새로운 쇼핑몰 앱 프로젝트 시작하기" |
| **`list_projects`** | **프로젝트 목록 조회:** 사용자가 접근할 수 있는 모든 프로젝트 리스트를 불러옵니다. (기본값: 본인 소유) | "내가 지금까지 만든 프로젝트 목록 확인" |
| **`get_project`** | **프로젝트 상세 조회:** 프로젝트 이름을 기반으로 특정 프로젝트의 세부 설정과 정보를 가져옵니다. | "'쇼핑몰_앱_V1' 프로젝트의 설정값 확인" |

---

## 📖 제2장. 스크린(Screen) 관리하기

프로젝트(스케치북) 안에 들어있는 개별 화면(페이지)들을 조회하는 명령어들입니다.

| **명령어 (함수명)** | **기능 설명** | **활용 예시** |
| --- | --- | --- |
| **`list_screens`** | **스크린 목록 조회:** 지정한 프로젝트 안에 포함된 모든 스크린의 목록을 보여줍니다. | "쇼핑몰 프로젝트 내의 전체 화면(로그인, 장바구니 등) 리스트 확인" |
| **`get_screen`** | **스크린 상세 조회:** 특정 스크린 하나를 지정하여 해당 화면의 상세한 디자인 및 코드 정보를 가져옵니다. | "쇼핑몰 프로젝트 중 '로그인 화면'의 세부 정보 확인" |

---

## 📖 제3장. AI 기반 화면 생성 및 디자인 (핵심)

텍스트 프롬프트를 이용해 AI에게 직접 디자인을 지시하는 심화 기능입니다.

| **명령어 (함수명)** | **기능 설명** |
| --- | --- |
| **`generate_screen_from_text`** | **텍스트로 새 화면 생성:** 텍스트 명령(프롬프트)을 바탕으로 프로젝트 내에 완전히 새로운 화면을 디자인합니다. |
| **`edit_screens`** | **기존 화면 수정:** 이미 만들어진 화면의 일부를 텍스트 명령을 통해 수정하거나 업데이트합니다. |
| **`generate_variants`** | **화면 시안(변형) 생성:** 완성된 화면을 바탕으로 다양한 버전(A안, B안 등)의 디자인 시안을 추가로 만들어냅니다. |

> **⚠️ [중요] 제3장 AI 명령어 사용 시 필수 주의사항**
> 
> - **대기 시간:** AI가 디자인을 생성하고 렌더링하는 데 **수 분의 시간**이 소요될 수 있습니다. 인내심을 갖고 기다려야 합니다.
> - **재시도 금지 (DO NOT RETRY):** 중간에 통신(네트워크) 에러가 발생하더라도, 서버 내부에서는 작업이 성공적으로 진행되고 있을 확률이 높습니다. 절대로 중복해서 명령어를 재실행하지 마시고, 나중에 `get_screen` 명령어로 결과물 생성을 확인하십시오.
> - **AI의 제안(Suggestions) 처리:** `generate_screen_from_text` 실행 결과로 AI가 추가적인 화면("이 화면도 같이 만들까요?")을 제안할 수 있습니다. 사용자가 이를 수락하면, 해당 제안을 프롬프트로 삼아 명령어를 한 번 더 실행하면 됩니다.
